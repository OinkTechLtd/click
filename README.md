# Click

Click — production-ready фронтенд-приложение на React + TypeScript + Vite для создания коротких ссылок с заданиями, таймером и милой уткой. Проект вдохновлён механикой сервисов вроде Linkify: автор вставляет финальную ссылку, добавляет задания, получает короткий URL, а посетитель должен выполнить задания или подождать на странице перехода.

## Возможности

- ✅ Современный стек: Node.js, Vite, React, TypeScript.
- ✅ Полностью без собственного бэкенда и базы данных.
- ✅ Генерация ссылки перехода с полезной нагрузкой внутри URL hash.
- ✅ Автоматический fallback между внешними API коротких ссылок: `is.gd`, `v.gd`, `TinyURL`, `CleanURI`.
- ✅ Если задания не указаны — посетитель ждёт 60 секунд по умолчанию.
- ✅ Если посетитель пытается скипнуть таймер или подтвердить задание слишком рано — показывается плачущая утка.
- ✅ При успешном прохождении — радостная утка и кнопка перехода.
- ✅ Лендинг, FAQ, документация, условия использования и политика конфиденциальности на русском языке.
- ✅ Обязательное принятие условий и политики при первом запуске.
- ✅ Готово для статического деплоя на GitHub Pages, Netlify, Vercel, Cloudflare Pages или любой CDN.

## Быстрый старт

```bash
npm install
npm run dev
```

Откройте адрес, который покажет Vite, обычно `http://localhost:5173`.

## Production build

```bash
npm run build
npm run preview
```

Папка `dist` содержит статическое приложение. Её можно опубликовать на любом статическом хостинге.

## Как это работает

1. Автор открывает `/create`.
2. Вставляет финальный URL.
3. При необходимости добавляет задания: Telegram-канал, сайт, страницу подписки, форму и т.д.
4. Click кодирует конфигурацию ссылки в URL вида `/#/go/<payload>`.
5. Приложение пробует сократить URL через API `is.gd`. Если провайдер недоступен, автоматически переключается на следующий.
6. Посетитель открывает короткую ссылку, ждёт таймер и выполняет задания.
7. После выполнения всех условий открывается финальная кнопка перехода.

## Важное ограничение браузера

Click не использует собственный сервер, поэтому браузер не может надёжно читать состояние загрузки чужого cross-origin сайта. Вместо этого реализована безопасная фронтенд-проверка:

- фиксируется факт открытия задания в новой вкладке;
- пользователь должен подождать заданное автором количество секунд;
- слишком раннее подтверждение считается попыткой скипа;
- блокировщики pop-up и невозможность открыть окно также приводят к ошибке прохождения.

Для строгой серверной проверки подписок, логинов или фактической загрузки внешних страниц нужен отдельный backend и интеграции с API соответствующих платформ.
