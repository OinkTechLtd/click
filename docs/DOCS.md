# Документация Click

## Назначение

Click создаёт интерактивные короткие ссылки. Автор ссылки может указать финальный URL, описание, таймер и список заданий. Посетитель получает доступ к финальному переходу только после выполнения условий.

## Архитектура

Click — статическое SPA-приложение:

- frontend: React + TypeScript;
- сборщик: Vite;
- маршрутизация: hash routes (`#/create`, `#/go/...`), чтобы приложение работало на GitHub Pages и любом static hosting;
- хранение согласия и последних ссылок: `localStorage`;
- собственный backend: отсутствует.

## Shortener fallback

При генерации Click пробует провайдеров в порядке здоровья:

1. `is.gd`
2. `v.gd`
3. `TinyURL`
4. `CleanURI`

Если API возвращает ошибку, timeout или некорректный ответ, провайдер получает failure-score, а приложение автоматически пробует следующий. При успешном ответе score сбрасывается.

## Payload ссылки

Настройки перехода сериализуются в JSON, кодируются в base64url и добавляются в hash URL. Это позволяет работать без базы данных, но означает, что слишком большое количество заданий увеличит длину ссылки.

## Проверка заданий

В браузере нельзя гарантированно определить, загрузился ли внешний cross-origin сайт. Click реализует клиентскую модель контроля:

- задание открывается через `window.open`;
- если окно не открылось, прохождение блокируется;
- пользователь должен вернуться и нажать «Проверить» только после минимального времени;
- ранняя проверка помечается как попытка скипа.

## Деплой

```bash
npm install
npm run build
```

Опубликуйте папку `dist`. Для Vercel/Netlify обычно достаточно указать build command `npm run build` и output directory `dist`.
